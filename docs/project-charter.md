# Project Charter

## Project name
- Super-Template

## Purpose
- Provide a practical starting point for project planning, documentation, and AI-assisted execution.

## Problem statement
- Teams need a fast way to start with a usable documentation and governance baseline instead of rebuilding it each time.

## Goals
- Publish required planning and operations docs.
- Standardize templates for repeatable project setup.
- Establish governance for Copilot/agent automation.

## Non-goals
- Prescribe a specific application stack.
- Replace product-specific engineering decisions.

## Stakeholders
- Project maintainer(s)
- Engineering contributors
- Product and operations collaborators

## Success criteria
- Required docs and templates exist and are discoverable.
- Governance rules are explicit and reviewable.
- New contributors can follow the process without tribal knowledge.
